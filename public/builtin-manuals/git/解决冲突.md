## 解决冲突

代码合并或更新代码时经常会遇到冲突的情况，需要手动解决冲突后再提交。

#### 解决冲突的流程

1. 执行 `git pull` 把代码拉下来，git 会自动尝试合并
2. 编辑冲突文件，根据实际情况保留本地代码还是远端代码
3. 暂存文件并推送到远端

```bash
# 拉取最新代码（可能产生冲突）
git pull

# 手动编辑冲突文件，解决冲突标记
# <<<<<<< HEAD
# 本地代码
# =======
# 远端代码
# >>>>>>> branch-name

# 解决完冲突后暂存并提交
git add -A
git commit -m "fix: 解决合并冲突"
git push
```

#### 可视化冲突解决工具

- [meld](http://meld.sourceforge.net/install.html) - 开源的差异和合并工具
- [kdiff3](http://kdiff3.sourceforge.net/) - 三方合并工具
- 执行 `git mergetool` 命令会启动默认的合并工具
